# Objetivos

- Unificar la documentación principal consolidando los archivos `.md` sueltos de `src/` en sus ubicaciones definitivas.
- Mover guías arquitectónicas a `docs/architecture/` (ej. I18N-GUIDE.md, I18N-STATUS.md).
- Mover instrucciones de testeo a `docs/testing/` (ej. README-TESTS.md).
- Consolidar instrucciones de entorno, API, resolución de problemas y comandos Git dentro del `README.md` principal, eliminando los archivos locales redundantes en `src/`.
- Asegurar que la raíz del proyecto es la única fuente de la verdad para el setup general, alineado con las normas del sistema.
