# Objetivo Actualización README

## Objetivo
Unificar el archivo `README.md` principal del proyecto con los diferentes READMEs y archivos de documentación dispersos en el directorio `src/`, para crear una única fuente de verdad (Single Source of Truth) más clara y mantenible.

## Alcance
- Integrar contenido clave de archivos en `src/` (como `SETUP.md`, `CONFIGURACION-API.md`, `SOLUCION-CORS.md`, `SOLUCION-PROBLEMAS.md`, `INSTRUCCIONES.md`) dentro del `README.md` principal.
- Mover archivos de documentación específicos de arquitectura o testing a carpetas apropiadas (`docs/architecture/`, `docs/testing/`).
- Eliminar los archivos originales redundantes en `src/`.
- No modificar el código fuente de la aplicación, solo la documentación.

## Leyes Aplicadas
- Ley de la Single Source of Truth: Evitar información redundante y dispersa.
