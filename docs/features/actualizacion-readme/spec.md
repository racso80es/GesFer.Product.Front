# Especificación

- El archivo `README.md` de la raíz del proyecto debe contener la guía completa y única de: setup, configuración API, instrucciones de ejecución, problemas comunes (troubleshooting) y comandos de git específicos para pruebas.
- Los documentos `I18N-GUIDE.md` y `I18N-STATUS.md` situados actualmente en `src/` serán consolidados en `docs/architecture/i18n-guide.md`.
- El documento `README-TESTS.md` situado actualmente en `src/` será movido a `docs/testing/testing-guide.md`.
- Los archivos `.md` existentes en la carpeta `src/` (a excepción de los estrictamente necesarios si los hubiera, que no es el caso para la documentación general) serán eliminados para limpiar el proyecto.
