El plan consiste en las fases estándar del automatic_task:
1. Activación (bloqueo de tarea y creación de rama).
2. Documentación inicial (objectives, spec, clarify, plan, implementation).
3. Consolidación de testing docs (`docs/testing/`).
4. Consolidación de i18n docs (`docs/architecture/`).
5. Consolidación del README raíz e eliminación de archivos sobrantes en `src/`.
6. Testing y Verificación funcional (run_in_bash_session).
7. Finalización de tarea (movimiento a DONE y actualización de EVOLUTION_LOG).
8. Pre-commit y Submit.
