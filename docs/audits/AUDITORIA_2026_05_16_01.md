## 2. Pain Points (🔴 Críticos / 🟡 Medios)
Hallazgo: Uso de native `console.error` y `console.warn`
Descripción: Se ha detectado el uso generalizado de métodos nativos de consola (`console.error` y `console.warn`) en múltiples archivos clave, lo que elude el sistema de telemetría centralizado del proyecto (The Wall de observabilidad).
Ubicación:

- `src/contexts/auth-context.tsx`
- `src/auth.ts`
- `src/lib/api/auth.ts`
- `src/lib/utils/client-init.ts`
- `src/i18n.ts`
- `src/lib/config.ts`
- `src/lib/providers/query-provider.tsx`
- `src/tests/helpers/test-data-cleanup.ts`

## 3. Acciones Kaizen (Hoja de Ruta para el Executor)
Instrucciones exactas:
- Crear un script Node.js para reemplazar `console.error(msg, err)` por `logger.error({ error: err }, msg)` y `console.warn(msg, err)` por `logger.warn({ error: err }, msg)` en los archivos identificados.
- Asegurar que `import logger from '@/lib/logger';` se añade al principio de cada archivo si no existe.
- Definition of Done (DoD): El proyecto debe compilar (`npm run build`) y pasar los tests (`npm run test:all`) sin errores. No deben quedar ocurrencias de `console.warn` ni `console.error` nativos en el código fuente (exceptuando scripts de build o archivos irrelevantes de node_modules).
